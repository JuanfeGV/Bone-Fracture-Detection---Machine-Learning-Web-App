Fase 1 — Entendimiento del negocio

En esta fase se define con claridad el problema clínico y los objetivos del proyecto. El propósito esencial es entender cómo el sistema apoyará el diagnóstico de fracturas y qué beneficios espera obtener el usuario (radiólogos, médicos generales, centros de atención).

Objetivos de la fase

  • Identificar los actores clave (radiólogos, técnicos en imágenes, personal administrativo).
  • Definir criterios de éxito clínico y métricas de evaluación (p. ej. sensibilidad, especificidad).
  • Establecer requerimientos funcionales de la aplicación web (login, carga de imágenes, visualización de resultados).

Preguntas clave

  • ¿Qué tipos de fracturas y regiones anatómicas se analizarán (muñeca, cadera, etc.)?
  • ¿Qué nivel de confianza mínima se requiere para que una predicción sea considerada útil?
  • ¿Cómo se gestionará la privacidad y el consentimiento del paciente?

Resultados esperados

  • Documento con definición del problema, actores, objetivos y métricas de éxito.
  • Lista de requerimientos funcionales para la interfaz web y flujos de usuario.
  • Plan de validación clínica (pasos para pilotaje con profesionales).

Fase 2 — Entendimiento e ingeniería de datos

Esta fase se centra en reunir, explorar y preparar los datos que se utilizarán para entrenar el modelo. Se prioriza trabajar con datasets confiables, garantizar anonimato y documentar cada transformación aplicada.

Actividades principales

  1. Recolección de datasets públicos y verificación de licencias y uso ético.
  2. Exploración inicial: formatos de imagen, resoluciones, relación entre casos positivos y negativos.
  3. Limpieza: eliminación de imágenes corruptas/duplicadas y etiquetado consistente.
  4. Preprocesamiento: conversión a escala de grises (si aplica), redimensionado (p. ej. 224×224), normalización y aumento de datos si es necesario.
  5. División del dataset en conjuntos de entrenamiento, validación y prueba.

Recursos y enlaces (datasets públicos)

  • Bone Fracture Dataset — Kaggle
  • Bone Fracture Detection — Kaggle

Consideraciones técnicas

  • Registrar metadatos: fuente, fecha de descarga, número de imágenes por clase y formato.
  • Control de calidad: revisar contraste, anotaciones superpuestas o marcas que afecten el entrenamiento.
  • Privacidad: trabajar con datos anonimizados y documentar el manejo ético de la información.

Resultados esperados

  • Dataset limpio y estructurado listo para modelado (carpetas /labels/ o CSV con rutas y etiquetas).
  • Notebook o script documentado con los pasos de preprocesamiento aplicados.
  • Informe breve con estadísticas del dataset (clases, balance, ejemplos representativos).
